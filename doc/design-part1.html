sAgent 详细设计文档

项目：自我进化多智能体个性化学习氛围编程网站  |  版本：v1.0  |  日期：2026-06-10

技术栈硬性约束：本系统前端与后端开发必须统一使用 Node.js（TypeScript）作为唯一开发语言，严禁引入任何其他编程语言（包括但不限于 Python、Java、C#、Go、Rust 等）。所有功能模块、服务接口、数据处理逻辑及 Agent 实现均需遵循此语言约束。唯一例外：代码执行沙箱内部可运行用户提交的任意语言代码。
文档范围：本文档基于需求分析 v2.0，对 sAgent 平台进行全面详细设计，涵盖系统架构、功能模块、业务流程、接口定义、数据结构、部署架构及安全设计。

属性说明

4.2 路径生成算法

BKT + RL 混合路径算法

贝叶斯知识追踪 (BKT)：估算每个知识点的掌握概率 P(known)

强化学习 (RL)：基于历史学习效果数据，优化知识点推荐序列

约束层：应用知识图谱的前置依赖约束

推荐知识点 = argmax(
    P(掌握|推荐) × 学习收益权重
    - P(挫败|推荐) × 挫折惩罚权重
    + 路径连贯性奖励
)
subject to: 前置知识已掌握

4.3 路径调整触发条件

触发条件检测方式调整动作响应时间
连续 3 题失败实时检测回退前置知识点巩固即时
单题耗时 > 3 倍平均实时检测降低后续难度、提供提示即时
连续 5 题通过实时检测加速推进、提升难度即时
知识掌握率 < 60%阶段评估路径重构、补充薄弱点下一阶段
学习停滞 > 3 天定时检测Mentor Agent 介入24h 内
遗忘曲线触发定时检测推送复习任务复习节点

6.2 认证流程

JWT + Refresh Token 认证

Access Token 有效期 15 分钟，Refresh Token 有效期 7 天。支持 GitHub/微信 OAuth 第三方登录。

// 认证流程
1. 用户登录 → 验证凭证
2. 签发 AccessToken (15min) + RefreshToken (7d)
3. 请求携带 AccessToken → API Gateway 验证
4. AccessToken 过期 → 用 RefreshToken 换发新 AccessToken
5. RefreshToken 过期 → 重新登录
