## 2. 设计目标

基于模版图片的布局，每个知识点的讲解应实现：

1. **图文并茂**：每个核心概念都配有 SVG 教学图，而非纯文字罗列
2. **中文讲解**：所有标题、说明、标注均为中文
3. **卡片化布局**：每个知识点独立一张卡片，内含"图+文+代码"
4. **暗色主题**：与氛围编程深蓝黑主题一致
5. **渐进深入**：从动机→概念图→代码→总结，符合认知规律

## 4. 数据结构扩展

### 4.1 新增 SVG 图字段

在 `LectureConcept` 中新增 `svgDiagram` 字段：

```typescript
// lecture-content.data.ts

export interface LectureConcept {
  title: string;
  content: string;
  /** ★ 新增：配套 SVG 教学示意图 */
  svgDiagram?: SvgDiagram;
}

/** SVG 教学图描述 */
export interface SvgDiagram {
  /** 图类型 */
  type: 'flowchart' | 'comparison' | 'structure' | 'state' | 'relation' | 'animation';
  /** 图标题 */
  title: string;
  /** SVG 文件路径（存放于 public/images/knowledge-svg/） */
  src: string;
  /** 图下方简要说明 */
  caption: string;
}
```

## 5. SVG 教学图分类与设计规范

### 5.1 六类教学图

| 图类型 | 适用场景 | 典型知识点 | SVG 视觉特征 |
|--------|---------|-----------|-------------|
| **流程图** | 执行流程、步骤、算法 | if/else、for循环、事件循环 | 矩形→菱形→矩形，箭头连接，true/false分支 |
| **对比图** | 概念对比、异同分析 | var/let/const、== vs === | 左右两栏/多栏表格，颜色区分，✅❌标注 |
| **结构图** | 层级、组成、分类 | 数据类型分类、DOM树 | 树形/嵌套矩形，父子连线 |
| **状态图** | 状态变化、生命周期 | Promise、组件生命周期 | 圆角矩形状态节点，箭头转换，颜色编码 |
| **关系图** | 关联、依赖、引用 | 原型链、作用域链 | 节点+连线，箭头指向，标签说明 |
| **动画演示图** | 运行过程、变化过程 | 变量赋值、函数调用栈 | 分步骤卡片序列，箭头流动，内存块变化 |

### 5.2 SVG 制图规范

```
画布尺寸:  780 × 360（概念面板内嵌尺寸，适配右侧内容区宽度）
字体:      system-ui, -apple-system, sans-serif
语言:      中文
配色:
  背景:    #0d1b2a（深蓝黑，与模版一致）
  主色:    #f97316（橙 — 重点、关键路径）
  辅色:    #3b82f6（蓝 — 信息、辅助元素）
  强调:    #a855f7（紫 — 特殊标注）
  成功:    #22c55e（绿 — true/正确/推荐）
  警告:    #eab308（黄 — 提示/pending）
  错误:    #ef4444（红 — false/错误/不推荐）
  线条:    #475569（灰蓝 — 边框、连线）
  文字:    #ffffff（主文字）、#94a3b8（副文字）、#64748b（辅助文字）
圆角:      矩形 8~12px，小标签 4~6px
阴影:      适度使用，不喧宾夺主
边框:      1~1.5px，半透明描边
文字大小:  标题 16~18px，正文 12~13px，标注 10~11px，代码 11px monospace
```

### 5.3 SVG 文件命名规范

```
/images/knowledge-svg/{知识点编号}-{简要名称}.svg

示例：
  /images/knowledge-svg/JS-001-var-let-const.svg
  /images/knowledge-svg/JS-001-data-types.svg
  /images/knowledge-svg/JS-003-if-else-flow.svg
  /images/knowledge-svg/JS-008-promise-lifecycle.svg
```

## 6. ConceptPanel 组件改造方案

### 6.1 核心改动

在 ConceptPanel 的"核心概念"区域，当 `concept.svgDiagram` 存在时，**在概念标题下方、正文之前**插入 SVG 图：

```tsx
{/* 核心概念卡片 */}
{lectureContent.concepts.map((concept, i) => (
  <div key={i} className="glass rounded-xl p-4 border-l-2 ...">
    {/* 概念标题 */}
    <h4>{concept.title}</h4>

    {/* ★ SVG 教学图 — 新增 */}
    {concept.svgDiagram && (
      <div className="my-3 rounded-lg overflow-hidden border border-white/[0.06]">
        <img
          src={concept.svgDiagram.src}
          alt={concept.svgDiagram.title}
          className="w-full h-auto"
        />
        <p className="text-[10px] text-white/30 px-3 py-1.5 bg-white/[0.02] border-t border-white/[0.04]">
          {concept.svgDiagram.caption}
        </p>
      </div>
    )}

    {/* 概念正文 */}
    <p className="text-xs text-white/45">{concept.content}</p>
  </div>
))}
```

### 6.2 改动范围

| 文件 | 改动内容 |
|------|---------|
| `lecture-content.data.ts` | `LectureConcept` 接口新增 `svgDiagram?` 字段 |
| `ConceptPanel.tsx` | 概念卡片区域插入 SVG 图片渲染 |
| `/public/images/knowledge-svg/*.svg` | 新增各知识点的 SVG 教学图文件 |

### 6.3 兼容性

- `svgDiagram` 为可选字段，不填则不显示图（向后兼容）
- 已有的 `richMedia` 字段继续保留，用于互动组件（类型图鉴、检测台等）
- 核心概念区域优先展示 SVG 图，`richMedia` 互动组件仍展示在概念区下方
