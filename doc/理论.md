# sAgent 算法建议与理论依据

---

## 一、知识追踪与能力评估

### 1. 贝叶斯知识追踪（BKT）

**理论基础**：Corbett & Anderson (1995) 提出的经典知识追踪模型

**核心参数**：
| 参数 | 含义 | 典型初始值 |
|------|------|-----------|
| P(L₀) | 初始掌握概率 | 0.3 |
| P(T) | 学习转移概率 | 0.1 |
| P(G) | 猜测概率 | 0.25 |
| P(S) | 失误概率 | 0.1 |

**建议**：
- 用 EM 算法从用户答题历史中拟合个体参数
- 局限：假设知识点独立，不建模知识点间依赖 → 需要配合知识图谱使用
- 改进方向：加噪 BKT（Noisy-or BKT）处理多知识点混合题目

### 2. 深度知识追踪（DKT）

**理论基础**：Piech et al. (2015) — "Deep Knowledge Tracing"

**架构**：LSTM/Transformer 编码答题序列，输出未来答题预测概率

**优势**：
- 自动捕获知识点间隐含依赖
- 处理变长序列，支持多特征输入（答题时间、错误类型等）

**建议**：
- v1 可用 BKT（可解释性强、数据需求小）
- v2 引入 DKT/SAKT（Self-Attentive Knowledge Tracing）提升精度
- 在线学习场景考虑用 E2E DKT 做增量更新

### 3. 项目反应理论（IRT）

**理论基础**：Lord (1980) — 心理测量学经典理论

**核心公式**：
```
P(正确) = c + (1-c) / (1 + exp(-a(θ - b)))
```
- θ：能力参数
- a：区分度
- b：难度参数
- c：猜测参数

**应用场景**：初始能力诊断的自适应测试（CAT）

**建议**：
- 实现 CAT（Computerized Adaptive Testing）：每题后更新 θ 估计，选择信息量最大的下一题
- 终止条件：标准误 SE(θ) < 0.3 或达到最大题数
- 用 MLE 或 EAP（期望后验）估计能力值

## 二、学习路径规划

### 4. 强化学习（RL）路径优化

**理论基础**：Sutton & Barto (2018) — 强化学习导论

**MDP 建模**：
| 组件 | 定义 |
|------|------|
| 状态 S | 用户当前知识掌握向量 + 学习风格 + 情绪状态 |
| 动作 A | 推荐下一个知识点/练习 |
| 转移 T | 用户学习后的知识状态变化 |
| 奖励 R | 答题正确 +1，首次通过 +0.5，超时 -0.1，放弃 -0.3 |

**算法选择**：
| 算法 | 适用场景 | 优势 |
|------|----------|------|
| Multi-Armed Bandit (MAB) | 探索 vs 利用平衡 | 实现简单、在线学习 |
| Thompson Sampling | 推荐系统 | 天然探索机制 |
| PPO/SAC | 复杂路径规划 | 处理连续状态空间 |

**建议**：
- v1 用 **Thompson Sampling** 做推荐（数据效率高、实现简单）
- 路径约束：用 DAG 拓扑排序保证前置知识先学
- 奖励函数设计：
```
R = α·正确率 + β·首次通过奖励 + γ·学习效率 + δ·挫败惩罚
```

### 5. 多臂老虎机（MAB）用于 A/B 测试

**理论基础**：假设检验 + 在线学习

**推荐算法**：
- **Thompson Sampling**：适合 Agent 策略进化
- **UCB1**：适合内容推荐
- **LinUCB**：考虑用户特征的上下文老虎机

**建议**：
- Evolution Agent 用 Thompson Sampling 管理策略实验
- 比传统 A/B 测试效率高 2-3x（自动分配更多流量给好策略）

## 三、个性化推荐

### 6. 混合推荐算法

**理论基础**：Burke (2002) — 混合推荐系统

**推荐架构**：
```
┌─────────────────────────────────────────┐
│           混合推荐引擎                    │
├─────────────┬─────────────┬─────────────┤
│  协同过滤    │  内容特征    │  知识图谱    │
│  (CF)       │  (CBF)      │  (KG)       │
├─────────────┴─────────────┴─────────────┤
│           加权/切换/级联融合              │
└─────────────────────────────────────────┘
```

| 算法 | 适用推荐 | 公式/方法 |
|------|----------|-----------|
| User-based CF | 学习伙伴推荐 | 余弦相似度找相似用户 |
| Item-based CF | 题目推荐 | 项目相似度矩阵 |
| Content-based | 知识点推荐 | TF-IDF/Embedding 匹配 |
| Knowledge Graph | 前置知识推荐 | 图遍历 + 路径评分 |

### 7. 协同过滤 + 知识感知

**理论参考**：Khan et al. (2020) — Knowledge-Aware Collaborative Filtering

**建议**：
- 用 **LightGCN**（He et al., 2020）做图神经网络推荐
- 融合知识图谱边关系（前置、相关、进阶）作为图结构信息
- 冷启动问题：用 IRT 能力评估结果 + 学习风格问卷做初始推荐

## 四、间隔重复与复习调度

### 8. SM-2 算法

**理论基础**：Piotr Wozniak (1987) — SuperMemo 算法

**核心公式**：
```
下次复习间隔 = 间隔 × (1.8 - 0.8 × E + 0.1)
```
- E： easiness factor（难度因子，初始 2.5）
- E' = max(1.3, E + 0.1 - (5-q)×(0.08+(5-q)×0.02))
- q：用户评分 0-5

**建议**：
- 基础版用 SM-2 即可满足需求
- 进阶可用 **FSRS**（Free Spaced Repetition Scheduler，2023）— 基于机器学习优化的间隔重复
- FSRS 比 SM-2 复习效率提升约 20-40%

## 五、用户画像与聚类

### 9. 学习风格识别

**理论基础**：Kolb (1984) — 经验学习理论 / Felder-Silverman 模型

**学习风格维度**：
| 维度 | 两极 | 识别信号 |
|------|------|----------|
| 感知 | 视觉 vs 言语 | 图表查看频率、文档阅读时长 |
| 输入 | 实践 vs 理论 | 直接编码 vs 先读文档 |
| 处理 | 活跃 vs 反思 | 快速尝试 vs 深思熟虑 |
| 理解 | 序列 vs 全局 | 循序渐进 vs 跳跃式学习 |

**算法**：
- **K-Means / GMM** 聚类发现学习风格群组
- **贝叶斯更新**：每次交互后更新风格后验概率

### 10. 情绪识别

**理论基础**：Picard (1997) — 情感计算

**信号与推断**：
| 行为信号 | 推断情绪 | 置信度 |
|----------|----------|--------|
| 连续删除 > 5 次 | 挫败 | 高 |
| 长时间停顿 > 30s | 困惑 | 中 |
| 快速切换题目 | 焦虑 | 中 |
| 代码提交频率上升 | 自信 | 高 |
| AI 辅导使用增加 | 需要帮助 | 高 |

**建议**：
- 用 **隐马尔可夫模型（HMM）** 建模情绪状态转移
- 或用简单的 **规则引擎 + 贝叶斯更新**（v1 推荐，可解释性强）

## 六、Agent 进化与优化

### 11. Prompt 优化

**理论基础**：Zhou et al. (2022) — "Automatic Prompt Optimization"

**方法**：
| 方法 | 描述 | 适用场景 |
|------|------|----------|
| APE (Automatic Prompt Engineer) | LLM 自动生成+评估 Prompt 变体 | Tutor Agent Prompt 优化 |
| DSPy | 编程式 Prompt 优化框架 | 复杂 Agent Pipeline |
| MIPRO | 贝叶斯优化 Prompt 组合 | 多 Prompt 协同 |

**建议**：
- v1：人工 Prompt + A/B 测试（简单有效）
- v2：引入 APE 自动优化 Prompt 模板
- v3：用 DSPy 框架做端到端 Prompt 优化

### 12. 进化策略

**理论参考**：Rechenberg (1973) — 进化策略

**Agent 进化框架**：
```
种群 = {策略变体₁, 策略变体₂, ..., 策略变体ₙ}
适应度 = 教育效果指标（满意度 × 学习效率 × 掌握率）
选择 = 轮盘赌 / 锦标赛选择
变异 = Prompt 微调、参数调整
交叉 = 策略组合
```

**建议**：
- 用 **CMA-ES**（Covariance Matrix Adaptation Evolution Strategy）做连续参数优化
- 离散策略选择用 **Thompson Sampling**
- 每轮进化后保留 Top-K 策略，淘汰 Bottom 策略

## 七、代码评估与质量分析

### 13. 代码质量评估

**理论基础**：Halstead (1977) 复杂度度量 + McCabe (1976) 圈复杂度

**评估维度**：
| 维度 | 指标 | 算法 |
|------|------|------|
| 正确性 | 测试通过率 | 单元测试执行 |
| 复杂度 | 圈复杂度 | AST 分析（Tree-sitter） |
| 可读性 | 命名规范、注释率 | 规则匹配 + LLM 评估 |
| 安全性 | 漏洞检测 | Semgrep 规则 + LLM |
| 氛围匹配 | Vibe Match Score | Embedding 相似度 |

### 14. AI 代码幻觉检测

**理论参考**：Ji et al. (2023) — "Survey of Hallucination in NLP"

**检测策略**：
- **静态分析**：检查函数/库是否存在（npm registry 验证）
- **类型检查**：TypeScript 编译器验证
- **运行时验证**：实际执行检测
- **LLM 判别**：用另一个 LLM 检测幻觉
