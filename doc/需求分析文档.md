## 1. 项目概述

### 1.1 项目背景

当前在线编程教育市场存在以下核心痛点：
- **学习效率低**：67% 的自学者在 3 个月内放弃（Code.org 数据）
- **个性化不足**：传统平台无法根据用户能力动态调整学习路径
- **AI 教育渗透率有限**：2025 年仅 18%，但 78% 的编程学习者期望获得 AI 个性化辅导
- **氛围编程（Vibe Coding）兴起**：一种以"说意图、定氛围、控结果"为核心的新编程范式缺乏系统化教学平台

### 1.2 项目目标

构建全球领先的自我进化 AI 驱动氛围编程学习平台，核心目标：

| 目标维度 | 量化指标 |
|----------|----------|
| 学习效率 | AI 自适应路径让学习效率提升 3 倍，同等技能达成时间缩短 60% |
| 学习体验 | 氛围编程沉浸式编码，编码到反馈 ≤ 3 秒 |
| 学习效果 | 知识掌握率 ≥ 85% |
| 学习陪伴 | 7×24 智能体全天候辅导，满意度 ≥ 4.0/5 |
| 学习动力 | 游戏化成就体系，30 日留存率 ≥ 30% |

### 1.3 项目范围

**核心范围**：
- 自我进化多智能体系统（10 个专业化 Agent）
- 个性化学习路径系统（BKT + RL 混合算法）
- 氛围编程（Vibe Coding）学习内容体系（8 大知识模块）
- 用户学习行为数据采集与分析
- 在线代码编辑与执行环境

**不在范围内（v1）**：VR 编程、脑机接口、离线全功能

### 2.2 典型用户画像

**画像 A — 编程小白「小明」**
- 18-25 岁，非技术专业在校生，零编程经验
- 需求：从零学习 Python，清晰路径 + 即时反馈

- 25-32 岁，传统行业从业者，了解基础 HTML/CSS
- 需求：前端/全栈开发，3-6 个月达到就业水平
- 痛点：学习路线不清晰、缺乏项目实战、无法评估自身水平
- 成功标准：6 个月内通过前端开发岗位技术面试

**画像 C — CS 学生「小华」**
- 20-23 岁，计算机科学大三学生，掌握 C/Java 基础
- 需求：算法训练、项目实战、面试准备
- 成功标准：LeetCode 中等题通过率 ≥ 80%，获得大厂 Offer

**画像 D — 在职开发者「小张」**
- 28-35 岁，3 年经验后端开发者，精通 Java/Go
- 需求：学习 Rust/系统编程、AI 应用开发
- 成功标准：3 个月内用 Rust 完成一个 CLI 工具项目

## 3. 功能需求分析

### 3.1 自我进化多智能体系统

#### 3.1.1 智能体分层架构

| 层级 | 名称 | 包含 Agent | 核心职责 |
|------|------|-----------|----------|
| L0 | 元认知层 | Evolution Agent | 系统自我进化、策略优化、A/B 测试管理 |
| L1 | 编排调度层 | Orchestrator Agent | 意图识别、任务分解、Agent 调度、冲突解决 |
| L2 | 核心业务层 | Tutor、Evaluator、Path Planner、Debug、Code Review、Knowledge Graph、Interview、Mentor | 各专业化业务处理 |
| L3 | 基础能力层 | LLM Gateway、RAG Engine、Tool Executor、Memory Store | 基础 AI 能力支撑 |

#### 3.1.2 各 Agent 功能定义

**Orchestrator Agent（编排引擎）**
- 输入：用户消息 + 会话上下文 + 用户画像
- 输出：Agent 调度指令 / 聚合后的统一响应
- 决策参数：置信度阈值 0.7、最大并行 Agent 数 3、超时 30s

**Tutor Agent（智能辅导）**
- 能力：概念讲解、代码示范、苏格拉底式引导、多语言支持
- 进化路径：固定模板 → 上下文适配 → 个性化风格 → 自主教学策略

**Evaluator Agent（代码评估）**
- 能力：正确性检测、风格评估、复杂度分析、安全检测
- 进化路径：规则匹配 → LLM 语义评估 → 多维度综合评估

**Debug Agent（调试辅助）**
- 能力：错误定位、原因分析、修复建议、调试教学
- 进化路径：错误信息翻译 → 模式匹配 → 上下文感知 → 预防性教学

**Path Planner（路径规划）**
- 能力：能力诊断、路径生成、动态调整、目标分解
- 算法：BKT（贝叶斯知识追踪）+ RL（强化学习）混合

**Code Review Agent（代码审查）**
- 能力：规范检查、最佳实践、重构建议、安全漏洞检测
- 进化路径：基础规范 → 代码异味 → 最佳实践 → 架构级审查

**Knowledge Graph Agent（知识图谱）**
- 能力：知识点关联、前置检测、缺口识别、推荐学习
- 数据规模：Python 280+ 知识点、JavaScript 250+、Java 200+

**Interview Agent（面试模拟）**
- 能力：面试题生成、模拟面试、答案评估、薄弱点分析
- 进化路径：固定题库 → 难度自适应 → 岗位定制 → 面试教练

**Mentor Agent（学习导师）**
- 能力：策略建议、动机激励、时间管理、习惯养成
- 进化路径：定时提醒 → 行为识别 → 情绪感知 → 个性化激励

**Evolution Agent（进化引擎）**
- 能力：策略优化、效果评估、A/B 测试、参数调优
- 安全约束：一键回滚、灰度发布（1%→5%→20%→50%→100%）、效果下限 ≥ 95%

#### 3.1.3 Agent 通信协议（ACP/1.0）

```json
{
  "protocol": "ACP/1.0",
  "message": {
    "id": "msg-uuid",
    "from": "orchestrator",
    "to": "tutor",
    "type": "request|response|event|error",
    "payload": {
      "action": "explain_concept",
      "parameters": {},
      "context": { "userId": "", "sessionId": "", "userLevel": "beginner" }
    },
    "metadata": {
      "priority": "high|medium|low",
      "timeout": 30000,
      "retryPolicy": { "maxRetries": 2, "backoffMs": 1000 }
    }
  }
}
```

#### 3.1.4 协作模式

| 协作模式 | 适用场景 | 示例 |
|----------|----------|------|
| 串行协作 | 有明确依赖关系 | Tutor 讲解 → Evaluator 评估 |
| 并行协作 | 独立子任务 | Evaluator + Code Review 同时评估 |
| 条件协作 | 需要判断 | 代码提交 → 评估 → 若失败 → Debug 介入 |
| 递归协作 | 复杂任务分解 | Path Planner 调用 Knowledge Graph 获取前置知识 |

### 3.2 个性化学习路径

#### 3.2.1 用户画像构建

| 画像维度 | 数据来源 | 更新频率 |
|----------|----------|----------|
| 基础属性（年龄、职业、教育背景） | 注册信息 | 注册时 |
| 学习目标（目标岗位、语言、时间） | 入职问卷 | 可随时更新 |
| 能力水平（编程、算法、项目能力） | 诊断测试 + 行为数据 | 实时更新 |
| 学习风格（视觉/听觉/实践偏好） | 问卷 + 行为推断 | 每月更新 |
| 行为特征（学习时段、时长、频率） | 行为日志 | 实时更新 |
| 情绪状态（挫折感、自信心、兴趣度） | 行为模式推断 | 每次交互 |

#### 3.2.2 初始能力评估方案

| 评估阶段 | 评估内容 | 题目数 | 时长 |
|----------|----------|--------|------|
| 逻辑思维 | 模式识别、逻辑推理 | 8 题 | 15 min |
| 编程基础 | 目标语言基础语法 | 10 题 | 20 min |
| 问题解决 | 编码题（简单-中等） | 3 题 | 30 min |
| 学习偏好 | 学习风格、节奏问卷 | 15 项 | 5 min |

采用 IRT（项目反应理论）自适应测试，动态调整题目难度。

#### 3.2.3 路径动态调整触发条件

| 触发条件 | 调整动作 | 响应时间 |
|----------|----------|----------|
| 连续 3 题失败 | 回退到前置知识点巩固 | 即时 |
| 单题耗时 > 3 倍平均 | 降低难度、提供提示 | 即时 |
| 连续 5 题通过 | 加速推进、提升难度 | 即时 |
| 知识掌握率 < 60% | 路径重构、补充薄弱点 | 下一阶段 |
| 学习停滞 > 3 天 | Mentor Agent 介入 | 24h 内 |
| 遗忘曲线触发 | 推送复习任务 | 复习节点 |

#### 3.3.2 交互模式

| 模式 | 描述 | 用户操作 | AI 行为 |
|------|------|----------|---------|
| 氛围描述式 | Vibe Coding 核心模式 | 输入氛围关键词+功能描述 | 生成匹配氛围的完整代码 |

| 氛围竞猜 | 反向训练 | 观察界面效果，反推描述 | 评分并给出优化建议 |
| 代码审查 | AI 代码评审训练 | 审查 AI 生成代码 | 提供评审清单和参考答案 |

#### 3.3.3 实时反馈机制

| 反馈类型 | 触发条件 | 延迟要求 | 呈现方式 |
|----------|----------|----------|----------|
| 语法错误 | 代码编辑时 | ≤ 500ms | 编辑器内联标注 |
| Lint 提示 | 代码编辑时 | ≤ 1s | 波浪线 + 悬浮提示 |
| 运行结果 | 点击运行 | ≤ 3s | 输出面板 |
| 测试结果 | 提交评测 | ≤ 5s | 测试面板 |
| AI 分析 | 代码提交后 | ≤ 3s (TTFT) | AI 面板流式输出 |
| 智能补全 | 暂停输入 | ≤ 300ms | 编辑器内联建议 |
| 氛围匹配度 | Prompt 提交后 | ≤ 2s | 氛围匹配评分 + 建议 |
| 幻觉预警 | AI 生成代码时 | ≤ 1s | 可疑代码高亮标注 |

## 4. 非功能需求

### 4.1 性能需求

| 最大内容绘制 LCP | ≤ 2.5s |

| 系统可用性 | ≥ 99.9% |

### 4.2 安全需求

| 安全域 | 威胁 | 防护措施 |
|--------|------|----------|
| 认证授权 | 未授权访问、Token 泄露 | JWT + RBAC + OAuth 2.0 |
| 数据传输 | 中间人攻击 | 全站 HTTPS/TLS 1.3 |
| 数据存储 | 数据泄露 | bcrypt 密码哈希、AES-256 加密 |
| 代码执行 | 沙箱逃逸 | Docker + gVisor 多层隔离 |
| AI 安全 | Prompt 注入、幻觉代码 | 输入过滤、输出审查 |
| API 安全 | DDoS、暴力破解 | 令牌桶限流、WAF |

### 4.3 可扩展性需求

| 组件 | 扩展方式 | 扩展能力 |
|------|----------|----------|
| 业务服务 | K8s HPA | 2 → 20 Pod 自动伸缩 |
| 代码沙箱 | 预热池 + 按需创建 | 50 → 500 并发执行 |
| 数据库 | 只读副本 + 分区 | 读 QPS 10x 提升 |
| 缓存 | Redis Cluster 分片 | 内存与 QPS 线性扩展 |

### 4.4 兼容性需求

- 浏览器：Chrome/Firefox/Safari/Edge 最新 2 版本
- 移动端：PWA 支持，核心流程可用
- 无障碍：WCAG 2.1 AA 标准

## 7. 典型用例

### 7.1 UC-001：新用户注册与路径生成

| 项目 | 内容 |
|------|------|
| 角色 | 新用户 |
| 前置条件 | 用户未注册，访问平台首页 |
| 后置条件 | 用户账户已创建，个性化学习路径已生成 |

**基本流程**：
1. 用户点击「注册」
2. 填写注册信息（邮箱/手机 + 密码）
3. 系统验证并创建账户
4. 用户选择学习目标
5. 完成能力诊断测试（IRT 自适应）
6. Path Planner 生成个性化学习路径
7. 用户确认路径，进入学习仪表盘

### 7.2 UC-002：氛围编程练习

| 项目 | 内容 |
|------|------|
| 角色 | 已登录用户 |
| 前置条件 | 用户已进入练习页面 |
| 后置条件 | 用户代码已评测，结果已展示 |

**基本流程**：
1. 选择练习题
2. 系统展示题目 + 代码编辑器
3. 用户选择交互模式（氛围描述式/Prompt 迭代/自由编码）
4. 系统实时提供语法检查 + Lint + 智能补全
5. 用户点击「运行」→ 沙箱执行
6. 运行出错 → Debug Agent 辅助
7. 提交评测 → Evaluator + Code Review
8. 展示评测报告 + AI 改进建议

### 7.3 UC-003：AI 辅导对话

| 项目 | 内容 |
|------|------|
| 角色 | 已登录用户 |
| 前置条件 | 用户在学习过程中遇到问题 |
| 后置条件 | 问题已解答，对话记录已保存 |

**基本流程**：
1. 用户点击 AI 助手按钮
2. 输入问题（文字/代码/截图）
3. Orchestrator 识别意图 → 调度对应 Agent
4. Agent 基于用户上下文生成回答
5. 系统流式展示 AI 回答
6. 用户追问或结束对话
7. 保存对话记录，更新知识状态

### 7.4 UC-004：系统自我进化

| 项目 | 内容 |
|------|------|
| 角色 | Evolution Agent（自动触发） |
| 前置条件 | 系统运行 ≥ 2 周，有足够反馈数据 |
| 后置条件 | 策略已优化或回滚 |

**基本流程**：
1. 分析全局反馈数据
2. 识别可优化维度
3. 生成策略变体
4. 创建 A/B 测试（1% 流量）
5. 运行实验，收集数据
6. 统计分析（p < 0.05, Cohen's d ≥ 0.2）
7. 效果显著 → 渐进放量；不显著 → 回滚
8. 记录进化日志

## 8. 数据结构概览

### 8.1 核心实体关系

```
users (1) ──── (1) user_profiles
users (1) ──── (N) submissions
users (1) ──── (N) learning_paths
users (1) ──── (N) ai_sessions
knowledge_points (1) ──── (N) exercises
exercises (1) ──── (N) submissions
```

### 8.2 核心表设计

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| users | 用户信息 | id, email, password_hash, role, subscription |
| user_profiles | 用户画像 | user_id, abilities, learning_style, behavior |
| knowledge_points | 知识点 | id, name, domain, prerequisites, dependents |
| exercises | 练习题 | id, type, difficulty, test_cases, vibe_prompt |
| submissions | 提交记录 | user_id, exercise_id, code, ai_evaluation |
| learning_paths | 学习路径 | user_id, stages, progress, knowledge_state |
| ai_sessions | AI 会话 | user_id, type, agents_used, user_rating |
