# JS-001「变量与数据类型」图文并茂教学重新设计方案

## 一、现状诊断

### 1.1 当前教学结构

| 模块 | 内容 | 媒介形式 |
|------|------|----------|
| motivation | 2行文字 | 纯文本 |
| concepts × 3 | 列表式文字 | 纯文本（含少量 emoji） |
| codeExamples × 2 | 代码块 + 一句解释 | 代码 + 纯文本 |
| summary | 4条要点 | 纯文本 |
| tips × 5 | 警告/建议 | 纯文本 |
| thinkQuestions × 3 | 思考题 | 纯文本 |

### 1.2 核心问题

1. **纯文本灌输**：所有概念以「列表 + 段落」呈现，没有视觉层次，学习者需要自行脑补抽象关系
2. **概念扁平化**：「7 种原始类型」只是 1-7 编号罗列，类型之间的关系（原始 vs 引用、大小、用途）完全没有可视化
3. **代码缺乏场景感**：代码示例是脱离上下文的「演示代码」，没有真实项目场景代入
4. **缺少交互反馈**：学习者只能「读」，无法「点、拖、试」，概念停留在记忆层面而非体验层面
5. **var/let/const 对比只靠文字**：作用域、提升、重新赋值这些最适合用动画/图示的概念，只用了文字描述
6. **null vs undefined 混淆**：这两个概念的区别是初学者最大痛点，但只有一段代码，缺少对比图/决策树

## 二、重新设计理念

### 2.1 教学设计原则

| 原则 | 说明 | 实现方式 |
|------|------|----------|
| **先感知后定义** | 先让学习者「看到」变量是什么，再给术语 | 开场动画：变量 = 贴了标签的盒子 |
| **对比即学习** | 并排展示差异，而非依次描述 | var/let/const 三栏对比卡片 |
| **类比具象化** | 用生活类比降低认知门槛 | 数据类型 = 不同形状的容器 |
| **即时可试** | 每个概念都附带可运行的小实验 | 嵌入式代码沙箱 |
| **错误先行** | 先展示常见错误，再给正确做法 | 「找 Bug」互动环节 |
| **渐进揭示** | 信息分步展示，而非一次性全给 | 手风琴/步骤式展开 |

### 2.2 多媒体层次模型

```
┌──────────────────────────────────────────┐
│  Layer 5: 实战挑战    — 项目级任务       │  ← 终极目标
│  Layer 4: 互动实验    — 代码沙箱/拖拽    │  ← 动手验证
│  Layer 3: 动画演示    — 作用域/执行流     │  ← 动态理解
│  Layer 2: 图表对比    — 对比卡片/决策树   │  ← 结构化认知
│  Layer 1: 故事引入    — 生活类比/场景     │  ← 建立直觉
└──────────────────────────────────────────┘
```

### 3.5 代码示例重构：场景代入式代码（真实项目场景 + 分步执行动画）

**当前问题**：代码是脱离上下文的 demo 代码，如 `const appName = "sAgent"`。

**重构方案**：用一个完整的小型项目贯穿，代码随场景演进。

### 3.6 null vs undefined 专题（决策树 + 对比图）

**设计**：这是初学者最混淆的部分，需要专门的视觉对比

```
┌───────────────────────────────────────────────────────────┐
│  🔍 null vs undefined 深度辨析                              │
│                                                             │
│  ┌─ undefined ────────────┐  ┌─ null ──────────────────┐  │

│  │ typeof: "undefined"    │  │ typeof: "object" ← Bug! │  │
│  │ 真值性: falsy          │  │ 真值性: falsy           │  │
│  │                         │  │                         │  │
│  │ 🎯 什么时候出现：       │  │ 🎯 什么时候使用：       │  │
│  │ · 变量声明未赋值         │  │ · 函数返回「找不到」    │  │
│  │ · 函数参数未传入         │  │ · 对象属性初始为空      │  │
│  │ · 对象不存在的属性       │  │ · 清空一个引用          │  │
│  └─────────────────────────┘  └─────────────────────────┘  │
│                                                             │
│  🌳 实战决策树：我该用哪个？                                 │
│                                                             │
│       ┌──────────────┐                                      │
│       │ 需要表示"空"？│                                      │
│       └──────┬───────┘                                      │
│              │                                              │
│        ┌─────┴─────┐                                        │
│        ↓           ↓                                        │
```

## 七、教学效果对比

| 维度 | 当前方案 | 重新设计方案 |
|------|----------|--------------|
| **信息呈现** | 纯文本列表 | 动画 + 对比图 + 图鉴 + 思维导图 |
| **学习方式** | 被动阅读 | 猜测验证 + 找Bug + 沙箱实验 |
| **代码教学** | 孤立 demo | 真实项目场景 + 分步演进 |
| **概念对比** | 依次文字描述 | 并排视觉对比 + 决策树 |
| **错误学习** | 警告文本 | 互动找 Bug + 即时反馈 |
| **知识留存** | 靠记忆 | 靠体验（猜、点、拖、试） |
| **完成动机** | 无 | 记分 + 解锁 + 进度感 |
| **认知负荷** | 高（大量文字一次性呈现） | 低（分步揭示 + 视觉化） |

> **核心理念**：好的教学不是把知识「写清楚」，而是让学习者「经历清楚」。从「我知道」到「我体验到」，学习才能真正发生。
