### 2.2 Section ① 作用域范围对比

**设计思路**：左右并列，var 在左（红色危险），let/const 在右（蓝色安全）

**var 侧（左）**：
```
function fn() {
  ┌─ if (true) { ──────────────┐
  │    var x = 1;               │  ← var 声明
  └─────────────────────────────┘
  console.log(x); // 1 ✅      ← x 泄漏到 if 外面！
}
```
- 用红色虚线框标注函数作用域范围
- if 块内高亮 var 声明
- 关键：x 在 if 外仍可访问 → 红色警告

**let/const 侧（右）**：
```
function fn() {
  ┌─ if (true) { ──────────────┐
  │    let a = 1; const b = 2; │  ← let/const 声明
  └─────────────────────────────┘  ← a, b 在此销毁
  console.log(a); // ❌ ReferenceError
}
```
- 用蓝色虚线框标注块级作用域范围
- if 块闭合处标注"a, b 在此销毁"
- 关键：a 在 if 外报错 → 绿色安全标记

**视觉差异**：var 侧红色渐变背景 + 红色边框；let/const 侧蓝色渐变背景 + 蓝色边框

### 2.3 Section ② 变量提升 vs TDZ

**var 的变量提升（左）**：

分三层展示：
1. **你写的代码**（灰色）：
   ```js
   console.log(x);  // undefined
   var x = 5;
   ```
2. **引擎实际执行顺序**（红色高亮）：
   ```js
   var x;           // ← 提升到顶部
   console.log(x);  // undefined (不报错!)
   x = 5;           // ← 赋值留在原位
   ```
3. **警告**：⚠️ 声明前访问不报错，值是 undefined — 隐患源头！

**let/const 的 TDZ（右）**：

分三层展示：
1. **你写的代码**（灰色）：
   ```js
   console.log(y);  // ❌ ReferenceError!
   let y = 5;
   ```
2. **TDZ 时间线**（核心图解）：
   ```
   作用域开始                    作用域结束
   ├──── 🚫 TDZ 禁区 ────┤──── ✅ 可安全访问 ────┤
                            ↑
                         let y = 5
   ```
   - 红色区域 = TDZ 禁区（声明前访问报错）
   - 绿色区域 = 安全访问区
   - 蓝色竖线 = 声明点
3. **安全说明**：✅ 声明前访问直接报错 — 强制先声明再使用

### 2.4 Section ③ 实战决策树

```
          ┌──────────────────────┐
          │  需要声明一个变量？   │
          └──────┬───────────────┘
                 │
        ┌────────┴────────┐
        │                  │
   ┌────┴────┐      ┌─────┴─────────┐
   │ var?    │      │ 值需要重赋值？ │

```

- 根节点：橙色圆角矩形
- var 分支：红色，直接打叉
- let 分支：蓝色，附示例 `let count = 0; count++;`
- const 分支：紫色，附示例 `const API_URL = "...";`

### 3.2 新增 SVG 在讲解流中的位置

**插入位置**：在 `richMedia.comparisons[0]`（文字对比卡片）之后、`comparisons[1]`（null vs undefined）之前。

**实现方式**：在 `richMedia` 中新增一个 `svgDiagrams` 数组字段，或在 `comparisons[0]` 中添加 `svgSrc` 字段。

**推荐方案**：利用已有的 `analogy` 字段的 `image` 字段，将其从 emoji 升级为 SVG 路径：

```typescript
analogy: {
  title: '📊 let vs const vs var — 原理深度图解',
  image: '/images/knowledge-svg/JS-001-var-let-const-deep.svg',
  explanation: '左上：var 的函数作用域让变量泄漏到 if 块外；右上：let/const 的块级作用域将变量限制在 {} 内。左下：var 的变量提升让你在声明前"不小心"拿到 undefined；右下：let/const 的 TDZ 在声明前直接报错保护你。底部决策树：const 默认首选，let 仅在需要重赋值时使用，var 永远不用。',

```
