## 2. 设计目标

### 2.1 核心原则

1. **图讲原理，文讲细节** — SVG 图聚焦"为什么/怎么工作"的原理可视化，文字补充细节和边界情况
2. **风格统一** — 所有 SVG 遵循同一设计语言（暗色主题、配色体系、字体大小）
3. **渐进式实施** — 每个知识点独立可交付，先做 JS-002 作为示例验证
4. **信息密度适中** — 图不过于复杂，5 秒内能看懂核心信息

### 2.2 SVG 设计规范

```
尺寸：    viewBox="0 0 780 360"（宽屏横向）或 "0 0 480 520"（竖屏纵向）
背景：    深蓝渐变 #0d1b2a → #0a1628，圆角 12px
字体：    system-ui, -apple-system, sans-serif
标题：    16px bold white
正文：    11-12px，颜色分级见下
配色：    错误=#ef4444  警告=#eab308  成功=#22c55e
         信息=#3b82f6  强调=#a855f7  橙色=#f97316  青色=#06b6d4
卡片：    fill="#0f172a" stroke="#1e293b" 圆角 4-8px
连线：    stroke="#475569" width 0.8-1.2
```

### 2.3 SVG 图类型与适用场景

| type 值 | 含义 | 典型场景 | 示例 |
|---------|------|----------|------|
| `comparison` | 对比图 | A vs B 对比 | let vs const, == vs === |
| `structure` | 结构图 | 层次/分类/组成 | 数据类型树, React 组件树 |
| `flowchart` | 流程图 | 执行步骤/数据流 | 事件循环, Promise 链 |
| `state` | 状态图 | 状态机/生命周期 | React 生命周期, Promise 状态 |
| `relation` | 关系图 | 实体间关系 | 原型链, 模块依赖 |
| `animation` | 动画图 | 动态过程演示 | 冒泡捕获, 渲染更新 |

## 4. JS-002 示例设计（第一批实施内容）

### 知识点信息

- **nodeId**: JS-002
- **名称**: 运算符与表达式
- **motivation**: 运算符是程序"做计算"的方式。理解运算符，你才能让程序帮你算数、比较、做逻辑判断。特别是 === 和 == 的区别，是 JavaScript 最经典的面试题之一。

### concept 列表与 SVG 设计

#### Concept 1: 算术与赋值运算符

**SVG 设计**：`JS-002-arithmetic-assignment.svg`
- 类型：`structure`
- 布局：左右两栏
  - 左栏：算术运算符（+ - * / % **）配示例 `5 + 3 → 8`
  - 右栏：赋值运算符（= += -= *=）配等价转换 `x += 5 ⇔ x = x + 5`
- 底部：`++/--` 自增自减的前置/后置区别示意

#### Concept 2: 比较运算符与类型转换

**SVG 设计**：`JS-002-comparison.svg`
- 类型：`comparison`
- 布局：核心是 `==` vs `===` 的对比
  - `==`（松散相等）：先做隐式类型转换再比较，画一个"转换漏斗"示意
  - `===`（严格相等）：不转换，类型+值都必须相同
  - 下方列出经典反例：`[] == false → true`，`null == undefined → true`
- 底部：💡 口诀 "永远用 ===，除非你明确需要类型转换"

#### Concept 3: 逻辑运算符与短路求值

**SVG 设计**：`JS-002-logical-short-circuit.svg`
- 类型：`flowchart`
- 布局：短路求值流程图
  - `&&`：左边 falsy → 直接返回左值（短路），不执行右边
  - `||`：左边 truthy → 直接返回左值（短路），不执行右边
  - `??`：左边 null/undefined → 返回右值，否则返回左值
- 用流程图菱形判断框 + 箭头表示执行路径

## 5. 前端渲染效果（无需改动）

ConceptPanel 已有的 SVG 渲染逻辑：

```
┌─────────────────────────────────────────┐
│  1  三种变量声明方式              ▸     │  ← concept 折叠标题
├─────────────────────────────────────────┤
│  ┌───────────────────────────────────┐  │
│  │     SVG 图片（原理图）            │  │  ← <img> 渲染
│  │     var/let/const 对比表          │  │
│  └───────────────────────────────────┘  │
│  📌 默认用 const，需要变才用 let        │  ← caption
│                                         │
│  JavaScript 提供三种声明变量的关键字... │  ← 文字内容
└─────────────────────────────────────────┘
```

**交互**：每个 concept 是 `<details>` 折叠面板，有 `svgDiagram` 的 concept 默认展开（`open={!rich || !!concept.svgDiagram}`）。

## 7. 风险与缓解

| 风险 | 缓解 |
|------|------|
| SVG 手写耗时 | 遵循模板化布局（对比图/结构图/流程图三套模板），减少重复设计 |
| SVG 文件体积 | 手写 SVG 代码精简，单个文件 < 5KB |
| 后端重启编译 | Nest watch 模式自动重编译，无需手动重启 |
| 移动端 SVG 缩放 | SVG 用 viewBox 自适应，ConceptPanel 已设 `w-full h-auto` |
