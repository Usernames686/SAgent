# 氛围编程（Vibe Coding Learning System）完整重设计

## 一、当前问题分析

### 1.1 现状
- **氛围编程页面**：只是一个"描述氛围→生成组件"的代码生成器，根本不是学习系统
- **学习路径页面**：所有数据硬编码（STAGES 常量写死在前端），不是从后端获取的
- **Agent 系统**：虽然定义了多个 Agent，但实际只用了 Orchestrator → Tutor 单向调用，没有真正的多 Agent 协作学习流程
- **知识点系统**：KnowledgePoint 实体存在但没有任何初始化数据
- **个性化**：完全没有——不跟踪用户学习进度、不调整难度、不推荐内容

### 1.2 核心差距
```
当前：      用户 → [选择模板 → 提交 → 生成代码] → 结束
           
目标：      用户 → [评估能力 → 推荐知识点 → 多模态学习 → 
             Agent 协作辅导 → 练习 → 评估 → 调整路径 → 进化] →
             持续循环
```

## 二、重设计目标

### 2.1 核心理念
"氛围编程"应该是一个**从零到全栈的 Node.js + React 个性化学习系统**，而不仅仅是一个代码生成器。

### 2.2 三大支柱

| 支柱 | 说明 | 涉及的 Agent |
|------|------|-------------|
| **全知识点覆盖** | 从 JavaScript 基础到 React 高级，每个知识点都有多种学习方式 | Knowledge Graph, Tutor |
| **全学习方法** | 阅读 → 交互编码 → Quiz → 项目实战 → AI 对话 → 代码评审 → 调试 | All Agents |
| **全个性化** | 根据能力、进度、情绪、学习风格动态调整 | Path Planner, Mentor, Orchestrator |

### 3.2 知识点粒度
每个模块包含 **10-20 个知识点**，每个知识点：
- 有唯一的 `nodeId`
- 有前置依赖知识点列表
- 有 3 级难度（beginner/intermediate/advanced）
- 有对应的练习题（3-5 道）
- 有对应的 Quiz 题（2-3 道）
- 有对应的项目练习（可选）

## 四、多模态学习方法

### 4.1 六种学习模式

| 模式 | 说明 | 主导 Agent | 触发时机 |
|------|------|-----------|---------|
| **📖 知识阅读** | AI 讲解知识点，含代码示例 | Tutor | 首次学习 / 复习 |
| **💻 交互编码** | 在线代码编辑器 + 实时反馈 | Debug, Feedback | 练习环节 |
| **❓ Quiz 测试** | 选择题 / 判断题测试理解 | Evaluator | 学完知识点后 |
| **🎯 项目实战** | 综合项目练习 | Code Review, Evaluator | 模块结束时 |
| **💬 AI 对话** | 自由提问，苏格拉底式引导 | Tutor | 任何时候 |
| **🔍 代码评审** | AI 审查用户代码并给出建议 | Code Review | 提交代码时 |

## 五、多 Agent 协作学习引擎

### 5.1 Agent 职责矩阵

| Agent | 在学习中的角色 | 调用时机 | 输出 |
|-------|--------------|---------|------|
| **Orchestrator** | 路由学习意图、调度 Agent | 每次用户交互 | 路由决策 |
| **Tutor** | 知识讲解、示例演示、答疑 | 学习/提问时 | 教学内容 |
| **Evaluator** | Quiz 评分、代码评估、能力诊断 | 提交练习/Quiz | 评分+反馈 |
| **Debug** | 代码错误定位、修复建议 | 编码报错时 | 错误分析 |
| **Path Planner** | 知识点排序、路径动态调整 | 学习路径变化时 | 路径调整 |
| **Knowledge Graph** | 知识点关联、缺口检测、推荐 | 学完一个知识点后 | 推荐列表 |
| **Code Review** | 代码规范检查、最佳实践建议 | 提交项目代码 | 审查报告 |
| **Mentor** | 学习策略建议、动机激励 | 定期/用户倦怠时 | 激励+建议 |
| **Evolution** | 学习效果分析、策略优化 | 后台定时 | 优化参数 |

### 5.2 学习会话 Agent 协作流程

```
用户交互
   │
   ▼
Orchestrator.route() ─── 判断用户意图
   │
   ├─ "讲解概念" ──────────▶ Tutor.explain()
   ├─ "做练习"   ──────────▶ Evaluator.generateExercise()
   ├─ "代码报错" ──────────▶ Debug.analyze()
   ├─ "提交代码" ──────────▶ Code Review → Evaluator
   ├─ "下一步学什么" ──────▶ Knowledge Graph → Path Planner
   └─ "我学不进去了" ──────▶ Mentor.motivate()
```
