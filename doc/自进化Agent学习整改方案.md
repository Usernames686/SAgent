# 自进化Agent学习整改方案

> 目标：让0基础学习者轻松入门、真正学会、成为高手

## 一、现状诊断

### 已实现（骨架级）

| 模块 | 状态 | 问题 |
|------|------|------|
| 知识图谱 + 种子数据 | ✅ 115个知识点 | 无学习路径导航，学习者不知从哪开始 |
| 讲授内容(lecture) | ✅ 7步详细讲解 | 讲完就完，无互动练习衔接 |
| 练习系统(exercise) | ✅ 数据已有 | 无代码沙箱执行，练习无法提交验证 |
| Quiz题库 | ✅ 数据已有 | 答题后无解析推送、无错题本 |
| 学习路径 | ✅ 引擎存在 | 无自适应推荐，路径固定 |
| Agent编排 | ✅ 9个Agent | 大部分是空壳，未接入LLM |
| 进化引擎 | ✅ BKT+RL算法 | 无真实学习数据喂养，算法闲置 |
| 代码沙箱 | ✅ 服务存在 | 未与练习系统打通 |
| 学习进度 | ✅ 基础追踪 | 无可视化、无薄弱点识别 |

### 未实现（关键缺失）

| 功能 | 影响 |
|------|------|
| **新手引导流程** | 0基础用户打开页面不知做什么 |
| **自适应学习闭环** | 学-练-测-补 没有串联 |
| **实时代码执行反馈** | 练习写完代码无法运行验证 |
| **错题本+薄弱点追踪** | 做错就过去了，无法针对性补强 |
| **学习数据可视化** | 看不到进步，无成就感 |
| **Agent实时辅导** | Agent不响应，无法提问 |
| **社区互动** | 孤独学习，无交流激励 |
| **成就/徽章系统** | 数据有但未展示，无激励 |

---

## 二、整改总览（7个阶段）

```
阶段1: 新手引导（1周）         → 让人知道怎么开始
阶段2: 学-练-测闭环（2周）     → 让人真正动手学
阶段3: 自适应推荐（1周）       → 让人学对东西
阶段4: Agent辅导打通（1周）    → 让人有老师带
阶段5: 进化引擎激活（1周）     → 让系统越教越好
阶段6: 激励体系（3天）         → 让人坚持学
阶段7: 数据驱动优化（持续）    → 用数据持续改进
```

---

## 三、分阶段详细方案

### 阶段1：新手引导（优先级最高）

**目标**：0基础用户首次打开，3分钟内开始第一个知识点学习

#### 1.1 新手引导页面

- 文件：`apps/web/src/app/onboarding/page.tsx`
- 3步引导：
  1. 选择身份（0基础/有基础/想提升）
  2. 选择目标方向（前端/后端/全栈）
  3. 自动生成推荐学习路径首页

#### 1.2 学习仪表盘改造

- 文件：`apps/web/src/app/dashboard/page.tsx`
- 当前问题：打开后看到空列表
- 改进：
  - 大卡片"继续学习"（显示上次进度）
  - "今日任务"（系统推荐3个知识点）
  - "学习进度环"（总体完成率）
  - "下一步建议"（Agent推荐）

#### 1.3 知识点详情页增加"开始学习"按钮

- 当前：点进知识点只看内容
- 改进：顶部增加"开始学习"按钮，点击后：
  - 自动记录学习开始时间
  - 底部增加"下一步：练习"引导
  - 学完后标记完成

---

### 阶段2：学-练-测闭环（核心价值）

**目标**：每个知识点都有 讲授→练习→测验→补强 的完整闭环

#### 2.1 练习系统打通沙箱

- 当前：`exercise-scoring.service.ts` 用正则匹配检查代码
- 问题：正则无法验证代码是否真正能运行
- 改进方案：
  - 练习提交 → `sandbox.service.ts` 执行代码 → 对比输出
  - 3级验证：Level1正则初筛 → Level2运行验证 → Level3 AI评审
  - 文件改动：`exercise.controller.ts` 增加 `/submit` 端点

#### 2.2 练习页面交互改造

- 文件：`apps/web/src/app/dashboard/vibe/practice/page.tsx`
- 改进：
  - 左侧代码编辑器（Monaco Editor已有）
  - 右侧输出面板（运行结果/错误信息）
  - 底部"提交验证"按钮
  - 提交后显示3级验证结果+AI点评

#### 2.3 Quiz答题闭环

- 当前：题库数据完整，但答题后无后续
- 改进：
  - 答题后立即显示解析
  - 答错自动加入错题本（`error-review.service.ts`已有）
  - 错题本定时提醒复习（`spaced-repetition.service.ts`已有）
  - 文件改动：`vibe-learning.controller.ts` 增加 `/quiz/submit` 端点

#### 2.4 薄弱点识别

- 当前：`adaptive-learning.engine.ts` 存在但未接入
- 改进：
  - 每次答题记录对错 → 更新知识点掌握度
  - 掌握度<60%的知识点标记为"薄弱点"
  - 仪表盘展示薄弱点列表
  - 一键生成"补强计划"

---

### 阶段3：自适应学习推荐

**目标**：系统根据学习者水平自动推荐下一步该学什么

#### 3.1 学习路径引擎激活

- 当前：`learning-path.engine.ts` 有基础逻辑
- 改进：
  - 基于BKT模型计算每个知识点掌握概率
  - 推荐策略：掌握度最低的前置知识点优先
  - 每次完成学习后重新计算推荐

#### 3.2 推荐API

- 文件：`vibe-learning.controller.ts`
- 增加端点：
  - `GET /api/v1/vibe/recommendation` — 获取推荐
  - `GET /api/v1/vibe/weak-points` — 获取薄弱点
  - `POST /api/v1/vibe/feedback` — 提交学习反馈

#### 3.3 前端推荐组件

- 仪表盘首页"为你推荐"卡片
- 知识点详情页"学完推荐"
- 学习结束后"下一步建议"

---

### 阶段4：Agent辅导打通

**目标**：学习者随时可以向AI导师提问

#### 4.1 LLM网关激活

- 当前：`llm.gateway.ts` 存在，.env有讯飞星火配置
- 改进：
  - 确认LLM连接可用
  - 增加对话上下文管理（最近5轮对话）
  - 增加知识点上下文注入（自动附带当前知识点内容）

#### 4.2 导师Agent激活

- 当前：`tutor.agent.ts` 是空壳
- 改进：
  - 注入LLM网关
  - 支持对话式辅导（不是一次性回答）
  - 辅导风格：苏格拉底式提问引导，不直接给答案

#### 4.3 辅导UI

- 每个知识点详情页右下角"问AI"按钮
- 点击弹出对话面板
- 对话历史保存到`ai-session`

---

### 阶段5：进化引擎激活

**目标**：系统根据群体学习数据自动优化教学内容

#### 5.1 数据收集

- 当前：`evolution-engine.service.ts` 有AB测试框架
- 改进：收集以下数据
  - 每个知识点平均学习时长
  - 练习通过率
  - 错题分布
  - 学习路径跳出率

#### 5.2 自适应内容调整

- 知识点通过率<50% → 自动增加更多讲解/提示
- 知识点平均时长>2倍预期 → 标记为"难度偏高"，调整难度标记
- 常见错误 → 自动生成补充练习

#### 5.3 教学策略优化

- AB测试不同讲解风格（代码优先 vs 概念优先）
- 根据学习效果自动选择最优策略
- 每周生成教学效果报告

---

### 阶段6：激励体系

**目标**：让学习者有成就感，愿意持续学习

#### 6.1 成就系统

- 当前：`badge.entity.ts` 有数据模型
- 改进：实现以下徽章
  - 🌱 初学者 — 完成第一个知识点
  - 📚 学习达人 — 连续7天学习
  - 🔥 代码高手 — 10道练习全通过
  - 🏆 知识大师 — 完成整个学习路径
  - 🤝 助人为乐 — 回答3个社区问题

#### 6.2 学习进度可视化

- 知识图谱可视化（已掌握/进行中/未开始）
- 每周学习时长统计图
- 掌握度雷达图（按知识域分类）

#### 6.3 社区互动

- 当前：`community-post.entity.ts` 有模型
- 改进：
  - 学习笔记分享
  - 代码展示墙
  - 每周学习排行

---

### 阶段7：数据驱动持续优化

#### 7.1 关键指标

| 指标 | 目标 | 衡量方式 |
|------|------|----------|
| 首日留存 | >60% | 注册后次日仍登录 |
| 知识点完成率 | >70% | 开始学习的知识点完成比例 |
| 练习提交率 | >50% | 看完讲解后提交练习 |
| 7日留存 | >30% | 注册7天后仍活跃 |
| NPS | >40 | 用户推荐意愿 |

#### 7.2 埋点方案

- 页面访问（哪个知识点被看最多）
- 练习提交（通过率、重试次数）
- Agent对话（提问频率、满意度）
- 流失点（在哪个步骤离开）

---

## 四、技术实现原则

1. **JSON化架构**：所有数据存`.json`，TS只做逻辑和类型
2. **API优先**：先实现API端点，再接前端
3. **渐进增强**：每个阶段独立可用，不依赖后续阶段
4. **数据驱动**：每个功能都有数据埋点，用数据验证效果

## 六、验收标准

| 阶段 | 验收标准 |
|------|----------|
| 1 | 0基础用户3分钟内开始学习，不需要看文档 |
| 2 | 每个知识点有讲授+练习+测验，代码可运行验证 |
| 3 | 首页推荐3个"下一步"，基于掌握度计算 |
| 4 | 任意页面可向AI提问，30秒内得到有用回答 |
| 5 | 系统自动识别薄弱点，生成补强计划 |
| 6 | 学习者能看到进度、徽章、排名 |
| 7 | 每周有数据报告，指导内容优化方向 |
